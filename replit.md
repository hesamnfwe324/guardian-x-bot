# GUARDIAN X ULTIMATE Bot

A production-ready Telegram group management bot.

## Run Locally

```bash
cp .env.example .env
# Fill in BOT_TOKEN, DATABASE_URL, REDIS_URL, BOT_OWNER_ID

# Option A: Docker Compose (includes Postgres + Redis)
docker-compose up -d

# Option B: Direct Python
pip install -r requirements.txt
python scripts/migrate.py   # creates tables + seeds achievements
python -m bot.main          # starts in polling mode if WEBHOOK_URL is empty
```

## Deploy to Render.com

1. Push this folder to GitHub
2. New Blueprint → connect repo → Render reads `render.yaml` automatically
3. Set secrets: `BOT_TOKEN`, `BOT_OWNER_ID`
4. Click **Apply** — Render provisions Postgres, Redis, and deploys the bot

## Environment Variables

| Var | Required | Notes |
|-----|----------|-------|
| `BOT_TOKEN` | ✅ | From @BotFather |
| `DATABASE_URL` | ✅ | `postgresql+asyncpg://...` |
| `REDIS_URL` | ✅ | `redis://...` |
| `WEBHOOK_URL` | prod | Your Render app URL |
| `BOT_OWNER_ID` | ✅ | Your Telegram user ID |
| `WEBHOOK_SECRET` | auto | Generated by Render |
| `SENTRY_DSN` | optional | Error tracking |

## User Preferences

- Python + aiogram 3, SQLAlchemy 2 async
- Webhook mode on Render, polling for local dev
